Để **bao phủ toàn bộ tài liệu BRD/SRS + Use Case + API/UAT**, mình đề xuất bộ endpoint theo 4 nhóm prefix:

* `/erp/v1/*` → pull/push với ERP FAST
* `/wms/v1/*` → API nghiệp vụ nội bộ WMS
* `/integration/v1/*` → retry, outbox, đồng bộ, log
* `/reports/v1/*` → dashboard, báo cáo

## 3) Bộ tối thiểu nên chốt

Nếu bạn muốn chốt nhanh cho tài liệu, mình khuyên chia thành 3 lớp:

* **Lớp 1 – bắt buộc theo UAT:** 16 API lõi
* **Lớp 2 – bắt buộc để cover BRD/SRS:** các endpoint từ mục B đến K
* **Lớp 3 – nice to have:** dashboard/reporting

## 4) Gợi ý chuẩn tên endpoint

Để tài liệu đẹp và nhất quán, nên dùng:

* danh từ số nhiều cho resource: `/items`, `/shipments`, `/putaway-tasks`
* action cho nghiệp vụ đặc biệt: `/submit`, `/confirm`, `/review`, `/retry`, `/push`
* phân tách rõ inbound/outbound/integration/report

Mình có thể làm tiếp cho bạn **bảng mapping “Use Case / Module / Endpoint / Method / Request chính / Response chính”** để bạn đưa thẳng vào tài liệu.
